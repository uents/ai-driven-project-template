# 統合テスト仕様書

## テストケース一覧

### TC-INT-001: ユーザー作成フロー

| 項目 | 内容 |
|---|---|
| テストID | TC-INT-001 |
| テスト名 | ユーザー作成フロー |
| 事前条件 | DynamoDBテーブルが空であること |
| 入力 | `{"name": "田中太郎", "email": "tanaka@example.com"}` |
| 期待結果 | ユーザーがDynamoDBに保存され、UserCreatedイベントが発行される |
| 確認項目 | 1. HTTPステータス201 2. DynamoDBにレコードが存在 3. EventBridgeにイベントが発行 |

### TC-INT-002: メールアドレス重複チェック

| 項目 | 内容 |
|---|---|
| テストID | TC-INT-002 |
| テスト名 | メールアドレス重複チェック |
| 事前条件 | 同じメールアドレスのユーザーが既に存在すること |
| 入力 | `{"name": "鈴木花子", "email": "tanaka@example.com"}` |
| 期待結果 | 409 Conflictが返却される |
| 確認項目 | 1. HTTPステータス409 2. エラーコードがDUPLICATE_EMAIL |

### TC-INT-003: ユーザー取得フロー

| 項目 | 内容 |
|---|---|
| テストID | TC-INT-003 |
| テスト名 | ユーザー取得フロー |
| 事前条件 | 対象ユーザーがDynamoDBに存在すること |
| 入力 | `GET /users/{user_id}` |
| 期待結果 | ユーザー情報がJSON形式で返却される |
| 確認項目 | 1. HTTPステータス200 2. レスポンスにuser_id, name, emailが含まれる |
