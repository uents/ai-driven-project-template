# 統合テスト計画書

## 概要

本ドキュメントは、統合テストの計画を定義する。

## テスト範囲

| 対象 | 内容 |
|---|---|
| API → Lambda | API Gatewayからのリクエストが正しくLambda関数にルーティングされること |
| Lambda → DynamoDB | Lambda関数からDynamoDBへのCRUD操作が正しく動作すること |
| Lambda → EventBridge | Lambda関数からのイベント発行が正しく行われること |

## テスト環境

- ローカル環境: moto によるAWSサービスのモック
- ステージング環境: 実際のAWSリソースを使用

## スケジュール

| フェーズ | 期間 | 内容 |
|---|---|---|
| テスト設計 | - | テストケースの作成 |
| テスト実装 | - | テストコードの実装 |
| テスト実行 | - | テストの実行と結果確認 |

## リスクと対策

| リスク | 対策 |
|---|---|
| AWS サービスの仕様変更 | moto のバージョンを固定する |
| テストデータの汚染 | 各テスト実行前後にクリーンアップを行う |
