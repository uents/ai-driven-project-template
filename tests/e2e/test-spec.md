# E2Eテスト仕様書

## テストケース一覧

### TC-E2E-001: ユーザー作成 → 取得シナリオ

| 項目 | 内容 |
|---|---|
| テストID | TC-E2E-001 |
| テスト名 | ユーザー作成 → 取得シナリオ |
| 事前条件 | ステージング環境がデプロイ済み |
| 手順 | 1. POST /users でユーザーを作成 2. レスポンスからuser_idを取得 3. GET /users/{user_id} でユーザーを取得 |
| 期待結果 | 作成したユーザーが取得できる |
| 確認項目 | 1. POST: 201 2. GET: 200 3. 名前・メールが一致 |

### TC-E2E-002: 存在しないユーザーの取得

| 項目 | 内容 |
|---|---|
| テストID | TC-E2E-002 |
| テスト名 | 存在しないユーザーの取得 |
| 事前条件 | なし |
| 手順 | 1. 存在しないUUIDで GET /users/{user_id} を実行 |
| 期待結果 | 404 Not Found が返却される |
| 確認項目 | 1. HTTPステータス404 2. エラーコードがNOT_FOUND |
