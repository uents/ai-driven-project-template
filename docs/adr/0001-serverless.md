# ADR-0001: サーバーレスアーキテクチャの採用

## ステータス

承認済

## コンテキスト

本システムはユーザー管理機能を提供するWebサービスである。チームは少人数で構成されており、インフラの運用負荷を最小限にする必要がある。トラフィックは時間帯によって大きく変動する。

## 決定

AWS Lambda + API Gateway + DynamoDB を中心としたサーバーレスアーキテクチャを採用する。

## 理由

- サーバー管理が不要で、少人数チームでも運用可能
- 従量課金モデルによりコスト効率が良い
- トラフィックの変動に自動でスケールする
- AWSマネージドサービスにより高可用性を実現

## トレードオフ

- **メリット**: 運用コスト削減、自動スケーリング、高可用性
- **デメリット**: コールドスタートによるレイテンシ、ベンダーロックイン、ローカル開発の複雑化

## 代替案

| 選択肢 | 評価 |
|---|---|
| ECS Fargate | 柔軟性は高いが運用負荷が増加する |
| EC2 | フルコントロール可能だが運用負荷が大きすぎる |
| コンテナ + App Runner | サーバーレスに近いが、DynamoDBとの親和性でLambdaが優位 |
