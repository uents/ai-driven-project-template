# ビジネス要件

## 概要

本システムは、ユーザー管理機能を提供するマイクロサービスシステムです。

## ユースケース

### UC-001: ユーザー登録

- **アクター**: エンドユーザー
- **概要**: 新規ユーザーがシステムにアカウントを作成する
- **事前条件**: なし
- **事後条件**: ユーザーアカウントが作成され、確認メールが送信される
- **基本フロー**:
  1. ユーザーが登録フォームに情報を入力する
  2. システムが入力値を検証する
  3. システムがユーザーアカウントを作成する
  4. システムがユーザー作成イベントを発行する
  5. システムが確認メールを送信する
- **代替フロー**:
  - 2a. 入力値が不正な場合、エラーメッセージを表示する
  - 3a. メールアドレスが既に登録済みの場合、エラーメッセージを表示する

### UC-002: ユーザー情報取得

- **アクター**: エンドユーザー
- **概要**: ユーザーが自身のプロフィール情報を閲覧する
- **事前条件**: ユーザーが認証済みであること
- **事後条件**: なし
- **基本フロー**:
  1. ユーザーがプロフィール画面にアクセスする
  2. システムがユーザー情報を取得して表示する

## ビジネスルール

- BR-001: メールアドレスはシステム内で一意であること
- BR-002: ユーザー名は2文字以上50文字以下であること
- BR-003: パスワードは8文字以上で英数字記号を含むこと
